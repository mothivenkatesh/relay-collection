**Product Headline**: 

1. Agentic automations for India’s payment infrastructure  
2. India’s 1st Payments MCP built for Agentic Era

**The one-liner**: Cashfree Relay turns your payments stack into an Agent that runs payment workflows within your AI tools like Claude, ChatGPT, Gemini etc. so any merchant now can run RTO recovery, refund triage, just by typing. 

Cashfree, but as your teammate that already knows your orders, payouts, refunds and settlements and can act on them.

**The Shift(why now)**: Every company is racing to put AI into their dashboard. Relay does the opposite: it puts Cashfree inside the AI agent the merchant already uses. Indian merchants don’t want one more dashboard. They want to ask, “~~identify all RTO-prone COD orders from the last 7 days and send a UPI prepayment link on WhatsApp~~,” and get it done on the go. N8n owns the Indian operators automation mindshare. Razorpay Agent studio is another dashboard, but Cashfree runs inside your Claude as MCP-first agentic approach is how we close it.

**What Cashfree Relay is?**

1. **Cashfree MCP:** that exposes Cashfree primitives (orders, payments, refunds, payouts, settlements, subscriptions, payment links, customer data) as tools any MCP-compatible AI agent can call. Auth, scoping, and audit trails sit inside Cashfree's compliant infrastructure.  
2. **Pre-defined, Indian payment use case templates**: Working agent skills for the highest-leverage workflows in India: RTO/COD recovery, refund triage, fake-order detection, abandoned-cart recovery, daily reconciliation, payout orchestration. Drop in, point at your data, run.  
3. **No-code Workflow orchestrator**: Once the workflow gets generated within Claude Cowork/code, merchants get the workflow link of Cashfree Relay \- no-code workflow builder just like n8n, where merchants can track Agent runs and configure it required.  
4. **Native install in Claude Desktop:** Works out-of-the-box in Claude Cowork (no-code, for non-technical operators) and Claude Code (for developers and workflow builders).

**What makes Relay different**

1. **Open MCP, not a closed agent.** Razorpay's surface is a walled garden. Relay is an open MCP the merchant owns: works in Claude today, any MCP-compatible agent tomorrow. The merchant keeps the keys.  
2. **Developer experience.** n8n won Indian dev mindshare with depth, Relay wins by going one layer deeper, payments-native.  
3. **Secure by design.** Scoped, audited tool calls run inside Cashfree's existing PCI/SOC perimeter. The agent never sees raw payment data, only the answers it's allowed to. No glue code. No API key management. No compliance overhead.  
4. **The single sharpest claim:** Relay is the fastest path from "Cashfree merchant" to "Cashfree merchant with an AI ops teammate that understands Indian payments" measured in minutes, not sprints.

**Messaging Pillars:**

1. **Agent-native, not dashboard-bolted-on.** Cashfree, where your team already works with.  
2. **India-first templates.** RTO, COD, fake orders, refunds are solved on day 1\.  
3. **No-code for ops. Full-code for engineering.** Same MCP. One install.

**Outcomes we target**:

* **Time-to-first-automation:** under 10 minutes for any starter template.  
* **Workflows shipped per active merchant:** 3 in the first month.  
* **Adoption:** \~100 merchants in v1 \+ soft unveil; a few thousand by Phase 2 close.

**What to avoid saying**

* Don't call it a "workflow tool" or "Zapier for payments." It's an agent platform like Google Agent Studio, not a workflow builder.  
* Don't pitch it as a "Claude plugin" or "AI feature." It's infrastructure with templates.  
* Don't promise full autonomy on irreversible actions (refunds, payouts) — Relay supports maker–checker, and we lean into supervised autonomy.  
* Don't position against developers. Claude Code support means engineers extend Relay; they aren't the threat.  
* Don't lead with "AI." Lead with the merchant outcome (RTO recovered, refund cleared, payout shipped). Operator language wins over feature language.

# Rzp Agent Strategy

**Razorpay’s Agent Strategy**

Razorpay is not building an Agent stack. They are running a defense mechanism against 3 threats:

1. MDR compressions  
2. LLM disintermediation  
3. pre-IPO valuation pressure

And, using AI as the wrapper that ties the response together. The actual strategic move is replacing the point APIs payments business with outcome-based take-rate business that captures a % of recovered merchant revenue. Agent Studio is one of 4 layers in their Agent stack. Custom Agents on request in the enterprise services play that makes the unit economics work at the top end. The whole thing is wrapped in a platform rebrand to capture AI-native fintech valuation multiple before listing.

Their overall AI Strategy:

There are 3 market forces hitting Razorpay at the same time, and their response to address them all.

1. MDR compression \- UPI-killed card-margin in India. Regulated bands compressed traditional take rates further. The PG business was a declining margin with no recovery path on volume.  
2. AI disintermediation \- As AI Agents like ChatGPT claude become the primary operational interface, the payment processor risks becoming a commodity API call inside someone else’s Agent. Say a merchant asking Claude to make payment. Claude doesn’t care which processor sits underneath.   
3. Pre-IPO valuation arbitrage \- AI-native fintech multiples started running 3-5x ahead of MDR-arbitrage fintech multiples in 2025\. As it’s trying to go public, every quarter spent narrating “AI-native Payments Leader” to compound the valuation gap before lockup.

The strategy had to defend the MDR margin, prevent disintermediation, and tell the public-market story simultaneously. One product launch couldn’t do that. A complete platform rebrand can do that.

**The 4-layer Agent stack:**

Each one addressing the merchant lifecycle,

1. **Layer 1: AI-native merchant experience**, they built a conversational onboarding experience which takes 5 mins over 45 mins with traditional form filling. Conversational integration like, Ask Razorpay to integrate payments into your App. This is for every razorpay merchant, existing & new. The base idea is to turn every razorpay platform surface into an AI surface.  
2. **Layer 2: Agentic Payments stack**. Moved from embed payments into apps & website forms to directly into AI Agents like Chatgpt, Gemini, Claude etc.This is commerce merchants who need to be present where buyers now shop. It’s their disintermediation defense as they own the system of record.  
3. **Layer 3: Agent Studio**. Pre-built agents (Cart Abandonment, Subscription Recovery, Dispute Responder, RTO Shield, Cashflow Forecaster, Settlement Insights). Plus custom agents on request for enterprise. This is for mid-market and enterprise merchants who want operational AI without building it themselves. The take-rate replacement engine.  
4. **Layer 4: AI-native Builder Distribution**. Native Razorpay integrations within Replit, Cursor, Lovable, n8n. Prompt to payments setup (ex: setup payments into my ecommerce site). This is for the next-gen merchants before they become merchants, the developers building micro-saas, ai-native apps, indie products. Land them at build phase, this they do it along with Razorpay Rize & continuous AI hackathons with all popular AI Agents like elevenlabs, OpenAI, Claude etc.

Every press cycle calls out, “World’s first AI Agent Studio for Payments”. NPCI co-launches with OpenAI & Anthropic. Builder Day. Platform Rebrand Newsletter. Each one moves the public narrative further towards “"Razorpay is AI-native fintech, not legacy MDR-arbitrage fintech."

**Agent Studio \- Strategy:**

What is Razorpay Agent Studio? It’s a hosted runtime inside Razorpay’s UI where merchants can:

1. Customize and deploy pre-built AI Agents  
2. Build custom Agents for tailored workflows (no-code beta)  
3. At enterprise level, request custom Agents built by Razorpay’s solutions team

It’s built on top of the Claude Agent SDK. Available via “early access" with 8 listed Agents.

**Business model**:

1. **Per-recovery fees(5-10% of recovered revenue)**: Industry-standard for cart-recovery and dispute-response tooling globally. RTO Shield prevents ₹X of RTO loss → Razorpay takes 5%.  
2. **Per-loss prevention fees**: Where the workflow stops a loss before it occurs (RTO prevention, fraud prevention), Razorpay takes a cut of the prevented loss against an agreed baseline.  
3. **Subscription tiers**: For CFOs who want predictable line items. Subscription Recovery agent at ₹X/month flat fee. Predictable for budgeting. Lower upside than outcome pricing but lower friction.  
4. **Bundled-mdr enterprise contracts**: For top merchants, Razorpay can negotiate "we'll uplift your effective take rate by 0.x% in exchange for unlimited Agent Studio access." This compresses procurement to one negotiation. Hides the agentic margin inside the existing MDR conversation. Hardest to compete against because it's invisible.  
5. **Custom Agent services**: The enterprise services play. Razorpay's solutions team builds bespoke agents for top 50 merchants at:   
   1. Setup fee: ₹25-75 lakh per custom agent (cost-plus on solutions time)  
   2. Outcome revenue share: 5-10% of recovered/prevented value, in perpetuity  
   3. Maintenance retainer: ₹X lakh/quarter for tuning. 

This is services margin (40-60%) rather than software margin (70-85%) but the per-merchant LTV is the highest in the portfolio.

**Unit economics example**: Take a Zomato-tier merchant: ₹500 crore/month GMV, 60% transactions involve some logistics layer, 12% RTO equivalent on that segment. 

Without RTO Shield equivalent custom agent: Monthly RTO-segment loss: 

* ₹36 crore Annual loss: ₹432 crore With a 30% reduction:   
* Monthly recovery: ₹10.8 crore   
* Annual recovery: ₹130 crore 

Razorpay take at 5% of recovered: 

* Monthly: ₹54 lakh   
* Annual: ₹6.5 crore per merchant 

For comparison, Razorpay's MDR on the same merchant (assume 0.3% effective on ₹500 crore/month): 

* Annual MDR: ₹18 crore 

The custom RTO agent take is 36% of MDR from a single agent on a single use case. Stack three custom agents (RTO \+ Dispute \+ Cashflow) and the agent take exceeds MDR. Now, we multiply with 30 enterprise merchants, that’s roughly 500cr/yr revenue that doesn’t exist in other ways, this compounds into pre-IPO valuation conversation.
