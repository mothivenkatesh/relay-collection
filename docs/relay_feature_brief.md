# Cashfree Relay — Feature Brief
*(Mothi voice, v2. No em dashes.)*

| Feature Name | Cashfree Relay |
| :---- | :---- |
| **Team** | CREX |
| **Release Date** | Preview Beta. Released, not yet marketed. |
| **Description** | Cashfree Relay puts Cashfree's payments stack inside the AI tools merchants and developers already use (Claude, ChatGPT, Cursor, Gemini). It ships as three surfaces on shared infrastructure: a Cashfree MCP server that exposes orders, payments, refunds, payouts, settlements, subscriptions, and payment links as tool calls any MCP-compatible agent can invoke; pre-built Agent Skills that encode India-specific payment workflows as reusable instruction packs; and a Visual Workflow Builder inside the Cashfree dashboard for operators who want a canvas with scheduling, retries, run history, and human-in-the-loop approvals.<br><br>Built for three personas inside the same merchant account. The Operator (ops lead, founder's office, RevOps) runs templates without writing code. The Builder (developer, founding engineer) gets Cashfree as MCP tool calls inside Cursor or Claude Code. The Partner (agency, freelancer, system integrator) packages and resells verticalised templates across multiple merchant accounts. Most useful for D2C brands managing RTO and COD, NBFCs and schools running monthly collection cycles, and any merchant team currently exporting CSVs, chasing failed payments manually, or stitching Cashfree events to other tools by hand.<br><br>Generic automation tools (n8n, Zapier) require manual webhook setup, have no awareness of Indian payment lifecycle edge cases, and push PCI compliance overhead onto the merchant. Relay solves this by exposing Cashfree primitives as native, typed tool calls with idempotency, audit trails, and maker-checker enforcement built in. All agent-driven actions run inside Cashfree's existing PCI DSS Level 1 / SOC compliance perimeter. The agent proposes. A human approves. |
| **Key Highlights** | **Cashfree MCP, open not walled**<br>An MCP server exposing the full Cashfree payment lifecycle as tool calls. Works in Claude today, ChatGPT and Cursor next, any MCP-compatible agent after that. Auth, scoping, and 90-day audit trails run inside Cashfree's existing compliance infrastructure. No proprietary client required.<br><br>**India-first payment workflow templates**<br>Pre-built Agent Skills for the highest-volume Indian payment workflows: threshold alerts, failed-payment digests, transaction-to-Sheets sync, calendar events on payment success, RTO/COD recovery, abandoned-cart recovery, refund triage, and monthly collection cycles. First workflow runs in under 10 minutes.<br><br>**No-code Visual Workflow Builder**<br>A canvas inside the Cashfree dashboard for operators who prefer nodes over prompts. Scheduling, retries, run dashboards, and human-in-the-loop approvals included. Workflows generated in Claude can be promoted to the builder in one click.<br><br>**Human-in-the-loop on every irreversible action**<br>Read tools enabled by default. Write actions (refunds, payouts, customer messages) require explicit merchant configuration and default to maker-checker. Confidence-gated execution: if the agent drops below the merchant's set threshold, the action lands in a human queue with reasoning attached.<br><br>**Native Cashfree event triggers**<br>Typed triggers across the full payment lifecycle: Payment Success, Failed, Dropped, Refund Success, Cancelled, Dispute Created/Updated/Closed, Schedule. Correct idempotency and signature handling built in. No manual webhook mapping, no glue code. |
| **Impact** | 1. Merchants run first payment automation in under 10 minutes using a starter template, no integration work required<br>2. Operators stop exporting CSVs, chasing failed payments manually, and context-switching between Cashfree and other tools<br>3. Developers get Cashfree as tool calls inside their existing AI IDE, no separate SDK or proprietary client<br>4. Partners (agencies, freelancers, SIs) package and resell verticalised Relay templates across merchant accounts<br>5. Merchants inherit Cashfree's PCI/SOC compliance perimeter for all agent-driven payment actions, no additional compliance overhead |
| **Feature Availability** | Available as: (1) an MCP server installable in Claude Desktop, Cursor, ChatGPT, and any MCP-compatible agent; (2) Agent Skills installable via `npx @cashfreepayments/agent-skills add skills` into Cursor, Claude Code, VS Code Copilot, Gemini CLI, and other AI coding assistants; (3) a Visual Workflow Builder inside the Cashfree merchant dashboard. Current connectors: Google Sheets, Slack, Gmail, WhatsApp Business, Google Calendar, Cal.com, Zoho Invoice, Calendly, Google Gemini, OpenAI, Cashfree Payments, HTTP Request. |
