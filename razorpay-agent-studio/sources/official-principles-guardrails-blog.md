# Official Principles, Guardrails, and Merchant Control Blog

**Source:** [Razorpay Agent Studio: Principles, Guardrails, and Merchant Control](https://razorpay.com/blog/razorpay-agent-studio-principles-guardrails-and-merchant-control/)

**Published:** March 30, 2026

## Key Points

- Defines Agent Studio as a marketplace inside the Razorpay dashboard where merchants can install AI agents for specific operational work.
- Explains that agents use verified first-party data from merchant-connected systems rather than scraping or external inference.
- Mentions established connector examples such as Shopify, Shiprocket, Tally, and QuickBooks.
- Says several connectors are pre-enabled from day one, including Razorpay data, WhatsApp, SMS, email, ElevenLabs, and Shiprocket.
- Emphasizes merchant consent through connector authorization and scoped access.
- Describes validation and certification for marketplace agents before merchant availability.
- Notes continuous monitoring of agent communication patterns and performance.
- Covers platform guardrails such as action boundaries, compliance checks, and amount validations.

## Governance Themes

- Merchant control and authorization.
- Verified data sources.
- Scoped connector access.
- Agent certification.
- Traceability and monitoring.
- Data privacy, security, and compliance.

## Compliance and Security Notes

The blog states that Agent Studio operates inside Razorpay infrastructure, inherits Razorpay's secure payment environment, and is built on compliance infrastructure aligned with applicable data protection laws. It also notes Razorpay's SOC 2 certification and PCI DSS compliance as a payment infrastructure provider.
